为什么深度学习中要搞出 tensor 这种概念，而不是直接用 numpy 里的 array 呢？

就是某个深度学习患者, 觉得 Tensor 这个词时髦, 就直接来了一波语义污染, 深度学习里这两者实际没有任何本质区别.

这词被几个深度学习框架污染前, tensor library 是这么个情况:

老牌 tensor library 也是有 Tensor 和 Array, Array of Array, Array of Array of Array... 等等的区别的

Tensor 一定是 NDArray，但是反过来 NDArray 不是 Tensor.

你直接问一个 NDArray 是什么 Tensor, 这没有意义

NDArray 只是 Tensor 的一种表征, 而不是其本身.

一般科研工作者只会关心阶, 指标, 对称性, 基, 变换这些属性, 而不会去关心具体的值.

所以库也不会基于 NDArray 实现 Tensor, 顶多最后拟合的时候展开一下就好.

人怎么算张量表达式，库也就模拟人怎么算，不会吃饱了没事干把下标全部展开变成一大片。

另一方面当时也是有专门的 NDArray 库的, 因为 Array of Array of Array of XXX 效率太低了, 会提供 CSR 这种更加好的计算基质。

两种库各干各的, 其乐融融。

然后某个深度学习大聪明就开始发癫了, 没事干把不带任何特殊性质的 NDArray 叫 Tensor.

而自从大聪明污染了这个词, 现在的数学, 物理, 乃至数学物理工作者不问问自己的前辈已经搜不到什么正儿八经的 tensor 库了.
