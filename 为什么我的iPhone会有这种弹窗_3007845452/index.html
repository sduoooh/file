为什么我的iPhone会有这种弹窗?

什么鬼说法，又是0级短信又是安珀警报的…目前的高赞们都在胡说八道些什么，甚至还造出了一个“t0”的说法…这里面错误太多了。


这就是个“闪信”，Flash SMS，一种特殊的短信类型，具体来说就是在发送短信的底层通信PDU时指定为Message Class 0。虽然俗称叫“0级短信”，但这是个不正确的翻译。Message Class 0不是指0级，这里没有“级”这种有高低之分的概念，它只是个分类，用于指示由具体哪个接收方设备来处理消息。其他Class可参考DCS：

Data Coding Scheme - Message classes
（维基不能访问的话直接看GSM文档：GSM 03.38）

简单说，其他三个Class各自指定由手机程序或者SIM卡（SIM里的程序）或者Modem来处理消息。Class 0的意思就是不指定处理方，既然没指定处理方，那么这个短信就是人来处理最好，于是常见做法就显示给用户，用户看了，就完事了。

这里要插嘴说一句，所有这些处理方法都不是硬性规定。并不是说闪信一定要即显，实际上很多手机并不即显只是显示个通知。其他的Message Class也并不是说一定要由特定的接收方设备处理，现在有的手机直接忽略全部消息都由应用来处理，以前没有智能机的时候有的手机全都交给SIM卡处理。还有的手机只处理Message Class 2的信息，别的全都丢掉。因为GSM没有功能性认证，实际只有各国入网许可来管这些事，所以很多手机设计上处理这些不常用的功能时相当随便。

回到闪信本身的话题。因为没有设备处理只是给人看，所以不会存下来（当然，也不是一定不会存下来，具体看手机设计）。因为不会存下来，所以不会因为存储空间不足的原因导致拒收。普通短信如果你的手机存储空间不足是会拒收的，超过一定时间一般是24到72小时这条短信就没了。于是它适合的是广播一些信息。但因为不会存下来于是想再看一遍没有了，所以并不适合显示重要信息。

于是反过来，因为它并不适合显示重要信息，所以运营商一般不会对其做出特殊限制。因为一般没有限制，它又有可能即显的特殊性，所以很多广告商会选择发这种信息，赌的就是万一你手机会即显那效果极好；最差因为不占用户存储空间不会被拒收，所以保底结果是会被用户看到。但缺点就是因为不会存下来所以之后看不到，于是一些可能需要用户之后再看的广告没法用（比如给你留电话号码微信号之类的）。

最终综合利弊，就出现了一种比较常见的做法就是题主看到的这样的“打招呼”型广告，运气好就能强迫你看，混个脸熟。甚至很多这些闪信服务是运营商主动推广给厂商使用的，就是主打一个会即显、用户一定会看到。


说完闪信，再说很多提到国外的警报信息的那些为什么错。

题主看到的也不是什么Wireless Emergency Alerts信息。WEA不发短信。WEA信息和普通短信的主要差异是WEA信息会有标题说明信息的主题，比如“Emergency Alert”这样的。

官方FAQ在这里：Wireless Emergency Alerts (WEA) - Are WEAs text messages?

所有提到其他国家用于广播灾难、紧急状态信息的都是WEA消息，目前的主要做法是运营商Cell Broadcast，翻译为蜂窝广播或者基站广播。顾名思义，这种消息是由基站发出的广播信息，它不需要手机加入通讯网络，也就是说不需要你的手机激活在网，甚至不需要插卡就能接收。

其实这事想想就很容易明白，比如本地即将发生地震这种紧急通知必须要你的手机插着不欠费的SIM才能收到…那忘交钱的人怎么办，去外国旅游的人怎么办，从外国来本地玩的人怎么办…


总结一下，这个答案主要说的是以下内容：
1、这就是“闪信”，是一种设计为只给人看一次的短信。没有“0级”这样暗示优先级高的效果，从设计上也并不适用于紧急广播等重要信息使用。具体效果实现完全取决于具体手机软件设计，没有统一标准。
2、常见的推送包含AMBER Alert等紧急广播信息的机制是Cell Broadcast，是一种仅仅只要手机开机并在基站有效信号范围内即可使用的消息推送机制。题主看到的信息不是Cell Broadcast信息。


顺便多说一句，闪信这个称呼是亚洲电信产品圈搞的名字，Flash SMS是反向翻译回去的称呼，不存在GSM标准里。全屏即显的操作也是这么来的，是一种厂商联盟标准处理，并非普遍的处理方式。
